# Dataset — Synthetic Email-Response Pairs

## Overview

This directory contains the synthetic dataset used to train and evaluate the
AI Email Response System.  All data is **machine-generated** using the
Cerebras API (Gemma 4) — no real user data or PII is involved.

## Data Schema

| Column             | Type   | Description                                      |
| ------------------ | ------ | ------------------------------------------------ |
| `id`               | `str`  | UUID v4 — unique identifier for each pair        |
| `category`         | `str`  | One of 14 email categories (see below)           |
| `sender_type`      | `str`  | Who sent the email (customer, employee, …)       |
| `email`            | `str`  | The incoming email text                          |
| `reply`            | `str`  | The ideal human-like response                    |
| `urgency`          | `str`  | `low` · `medium` · `high` · `critical`           |
| `sentiment`        | `str`  | `positive` · `negative` · `neutral` · `mixed`    |
| `style`            | `str`  | Writing style used to prompt generation          |
| `word_count_email` | `int`  | Word count of the email                          |
| `word_count_reply` | `int`  | Word count of the reply                          |

## Categories (14)

| #  | Category           | Description                                   |
| -- | ------------------ | --------------------------------------------- |
| 1  | Customer Support   | General customer inquiries and help requests   |
| 2  | HR                 | Human resources, policies, onboarding          |
| 3  | Internal Team      | Cross-team / internal communications           |
| 4  | Sales              | Sales inquiries, pricing, demos                |
| 5  | Refund             | Refund requests and return processing          |
| 6  | Technical Issue    | Bug reports, outages, tech troubleshooting     |
| 7  | Scheduling         | Meeting scheduling, calendar coordination      |
| 8  | Product Questions  | Feature inquiries, product capabilities        |
| 9  | Complaint          | Formal complaints and escalations              |
| 10 | Thank You          | Appreciation and positive feedback             |
| 11 | Partnership        | Partnership proposals, collaborations          |
| 12 | Billing            | Invoice, payment, and billing issues           |
| 13 | Interview          | Interview scheduling and candidate comms       |
| 14 | Logistics          | Shipping, delivery, supply-chain queries       |

## Writing Styles (9)

`formal` · `informal` · `angry` · `happy` · `ambiguous` · `missing_info` ·
`multi_question` · `short` · `long`

## Sender Types (6)

`customer` · `employee` · `partner` · `manager` · `vendor` · `candidate`

## Generation Methodology

1. **Batched prompting** — 5 pairs per API call to the Cerebras API
   (Gemma 4, model `gemma-4-31b`).
2. **Combinatorial diversity** — the generator cycles through shuffled
   `(category × style × sender_type)` tuples to ensure broad coverage.
3. **Validation** — each pair is checked for minimum word counts, correct
   enum values, and structural integrity before being accepted.
4. **Checkpointing** — progress is saved to CSV every 50 pairs so that
   interrupted runs can be resumed.
5. **Retry logic** — transient API errors trigger up to 3 retries with
   exponential back-off.

## Ethical Considerations

- **Fully synthetic** — every email and reply is generated by an LLM.
  No real customer or employee data is used.
- **No PII** — names, order numbers, and dates are all fictional.
- **Bias awareness** — LLM-generated data can reflect biases present in
  the model's training set.  Review downstream model outputs carefully.
- **Not representative** — distribution of categories, urgency levels,
  and sentiments is driven by the generation schedule, not real-world
  email volume.

## Limitations

- Synthetic emails may lack the full nuance and messiness of real
  customer communications (typos, screenshots, forwarded threads).
- Category boundaries are artificially clean — real emails often span
  multiple categories.
- The dataset does not include attachments, images, or email metadata
  (headers, CC/BCC, threads).
- Reply quality is limited by the capabilities of the generating model.

## Regeneration

```bash
# From the project root
python -m dataset.synthetic_generation --count 2000 --batch-size 5 --output dataset/emails.csv
```
